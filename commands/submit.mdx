# submit

> Submission lifecycle tools; use `release run` to ship

The `submit` command is for lower-level App Store submission lifecycle work.
For the canonical App Store publish flow, use `asc release run`.

## Usage

```bash  theme={null}
asc submit <subcommand> [flags]
```

## Canonical paths

Use these commands deliberately:

* `asc release run` - canonical App Store publish path
* `asc submit preflight` - inspect submission readiness without mutating anything
* `asc submit status` - inspect submission or version review state
* `asc submit cancel` - cancel an active submission

`asc submit create` remains available only as a deprecated compatibility path
for older automation that already prepared the version and only needs the final
submission step.

## Submission lifecycle

### Preflight a submission

Check readiness before you ship:

```bash  theme={null}
asc submit preflight --app APP_ID --version VERSION_STRING
```

### Check status

Inspect a submission by submission ID or version ID:

```bash  theme={null}
asc submit status --id SUBMISSION_ID
asc submit status --version-id VERSION_ID
```

### Cancel a submission

Cancel an active review submission:

```bash  theme={null}
asc submit cancel --id SUBMISSION_ID --confirm
```

## Canonical publish workflow

Ship to the App Store with the release pipeline:

```bash  theme={null}
asc release run \
  --app APP_ID \
  --version VERSION_STRING \
  --build BUILD_ID \
  --metadata-dir ./metadata/version/VERSION_STRING \
  --confirm
```

That flow ensures the version exists, applies metadata, attaches the build,
validates readiness, and submits for review.

## Deprecated direct submission path

If you still maintain older automation that already prepared the version, the
deprecated compatibility path is:

```bash  theme={null}
asc submit create --app APP_ID --version VERSION_STRING --build BUILD_ID --confirm
```

Prefer migrating that workflow to `asc release run`.

## Common errors

### No build attached

```
Error: Version 1.2.0 has no build attached
```

**Solution:** Use `asc release stage` or `asc release run` to attach the build as
part of the release pipeline.

### Missing metadata

```
Error: Missing required metadata for version 1.2.0
```

**Solution:** Complete the metadata, then retry `asc release run` or re-check
with `asc submit preflight`.

### Already in review

```
Error: Version 1.2.0 is already in review
```

Use `asc submit status` to inspect the current submission before retrying.
