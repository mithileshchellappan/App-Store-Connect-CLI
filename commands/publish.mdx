# publish

> High-level publish workflows; prefer `publish testflight` for TestFlight

The `publish` command is the high-level publishing surface for TestFlight.
For App Store shipping, use `asc release run`.

## Usage

```bash  theme={null}
asc publish <workflow> [flags]
```

## Canonical paths

Use these top-level workflows deliberately:

* `asc publish testflight` - canonical high-level TestFlight publish path
* `asc release run` - canonical App Store publish path
* `asc release stage` - prepare an App Store version without submitting it

`asc publish appstore` remains available only as a deprecated compatibility path
for older IPA-first automation.

## Workflows

### TestFlight workflow

Upload and distribute to TestFlight:

```bash  theme={null}
asc publish testflight \
  --app APP_ID \
  --ipa /path/to/MyApp.ipa \
  --group "External Testers"
```

This workflow:

1. Uploads the build
2. Waits for processing
3. Distributes to specified beta groups

### App Store workflow

Ship to the App Store with the canonical release pipeline:

```bash  theme={null}
asc release run \
  --app APP_ID \
  --version 1.2.0 \
  --build BUILD_ID \
  --metadata-dir ./metadata/version/1.2.0 \
  --confirm
```

This workflow:

1. Ensures the App Store version exists
2. Applies metadata and localizations
3. Attaches the selected build
4. Runs readiness validation
5. Submits for review

## TestFlight flags

<ParamField path="--app" type="string" required>
  App ID for the build
</ParamField>

<ParamField path="--file" type="string" required>
  Path to IPA file
</ParamField>

<ParamField path="--groups" type="string">
  Comma-separated beta group names or IDs

  Example: `"External Testers,Internal Team"`
</ParamField>

<ParamField path="--wait" type="boolean">
  Wait for build processing to complete

  Default: `true`
</ParamField>

## Deprecated App Store compatibility path

If you still maintain an older IPA-first script, the deprecated compatibility
path is:

```bash  theme={null}
asc publish appstore \
  --app APP_ID \
  --ipa /path/to/MyApp.ipa \
  --version 1.2.0 \
  --submit --confirm
```

Prefer migrating that workflow to `asc release run`.

## Example workflows

### Publish to TestFlight

```bash  theme={null}
asc publish testflight \
  --app 123456789 \
  --ipa build/MyApp.ipa \
  --group "External Testers,Beta Team"
```

### Publish to App Store

```bash  theme={null}
asc release run \
  --app 123456789 \
  --version 1.2.0 \
  --build BUILD_ID \
  --metadata-dir ./metadata/version/1.2.0 \
  --confirm
```

### CI/CD integration

```yaml  theme={null}
# GitHub Actions
- name: Publish to TestFlight
  run: |
    asc publish testflight \
      --app ${{ secrets.APP_ID }} \
      --ipa MyApp.ipa \
      --group "External Testers"
```

## Error handling

If any step fails, the workflow stops and reports the error:

```
Upload failed: Invalid IPA file
```

You can retry individual steps using the lower-level commands:

```bash  theme={null}
asc builds upload --app 123456789 --ipa MyApp.ipa
asc builds add-groups --build-id BUILD_ID --group "External Testers"
```
